3. Products

To represent products as JavaScript objects, two classes Product and ProductException will be added to the products.js file.

3.1 Product Class

The Product class should store the following data:

{
    "uuid":"df2008a5-1c40-4dd1-9db7-8aacc03ae2fb",
    "name": "Banana",
    "description": "A delicious and healthy fruit",
    "imageUrl":"https://images.freeimages.com/images/large-previews/4ec/banana-s-1326714.jpg",
    "unit": "piece",
    "stock": 15,
    "pricePerUnit": 3.6,
    "category":"Fruit"
}

This class does not interact with the product array directly. Getters and setters with appropriate validations (throwing ProductException exceptions) are needed, such as not allowing empty strings, negative numbers, etc.

For the UUID, an exception will be thrown in the setter to allow internal use only. The following code will be used to auto-generate UUIDs:

function generateUUID() {
    return 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'.replace(/[xy]/g, d => {
        let r = Math.random() * 16 | 0;
        let v = d == 'x' ? r : (r & 0x3 | 0x8);
        return v.toString(16);
    });
}

Copy this code into the utils.js file.

3.2 Product: clean and createFromObject

The following static functions will be created:

  • cleanObject(obj): Cleans the received object by removing values that are not part of the Product class.
  • createFromObject(obj): Validates that the received obj has the properties of the Product class; if it has all properties including _uuid return the cleaned object (call cleanObject method ); if it does not have the _uuid, create a new product instance (using the Product constructor) and return this new object.

3.3 Product: toHtml functions

  • toHtmlRow(obj, propOrder): Converts the product information into an HTML row (<tr>...</tr>, without headers), returning a string. Show the properties in the order received in the propOrder array (array of strings). Only show the properties in the array. If the array is empty or undefined, show all properties.
  • toHtmlDiv(obj, hideProps): Converts the product information into an HTML similar to the integrated assignment 1, i.e., a div with the product information, using the bootstrap classes. The hideProps array indicates which properties will be hidden. If the array is empty or undefined, show all properties.
  • toHtmlList(function(obj)): non-static method that converts the current product information into an HTML template depending on the received function, returning a string. In other words, this method only executes the function received as an argument and returns the result. By default, it will use toHtmlDiv.

4. Data Handling Functions

In the data_handler.js file, we'll handle an array of products, which will serve as our official list of products for sale. We'll add the corresponding methods to:

  • Read (getProducts(), getProductById(uuid))
  • Create (createProduct(product)): call the createFromObject method from the Product class.
  • Update (updateProduct(uuid, updatedProduct))
  • Delete (deleteProduct(uuid))

Additionally, we'll optionally add a method for filtering. For this, we'll create filterProducts(query, intersection=true), where query is an object array with a format

[   {
        property: "propName",
        value: "propValue",
        operator: "operatorValue"
     },
     {
        property: "propName",
        value: "propValue",
        operator: "operatorValue"
     }
]

The result will be a list of products that match the query. If intersection is true, the result will be the intersection of all queries. If intersection is false, the result will be the union of all queries.

The operator can be ==, !=, >, <, >=, <=, contains, startsWith, endsWith

We'll also implement toHtmlList(list), which takes a list, converts each element to HTML using the map method, and returns a single string by joining all products. This method calls the toHtml method from the Product class with the Product.toHtmlDiv function as an argument. To test this method, you can use View.renderList function.

Another method to implement is toHtmlTable(list, propOrder), which takes a list and an array of properties to show in the table. It converts each element to HTML using the map method and returns a single string by joining all products. This method calls the toHtml method from the Product class with the Product.toHtmlRow function as an argument. To test this method, you can use View.renderTable function.

5. Shopping Cart

In the shopping_cart.js file, we'll add three classes: ShoppingCart, ProductProxy, and ShoppingCartException.

To reference the products added to our shopping cart, we'll use the ProductProxy class, which stores the product UUID and the quantity to purchase. You don't need to use getters and setters for this class.

Within our ShoppingCart class, we'll maintain two arrays: one for proxies and another for products, where we'll store a copy of the actual products to be purchased. We'll ensure that the user cannot directly modify the arrays; therefore, setters should throw an error (ShoppingCartException), and the getters return a copy of the corresponding array (_proxies or _products).

We'll add the following methods:

  • addItem(productUuid, amount): Adds a new ProductProxy to the cart or updates the quantity if the product already exists. If the amount is a negative value, throw an error. Inside the method, use the ProdList to get the product and add it to the product array.
  • updateItem(productUuid, newAmount): Updates the quantity of the corresponding product. If the new value is negative, an error is thrown. If it's zero, the product is removed from the cart. If it's greater than zero, the quantity is updated. Keep updated on both the proxies and product arrays.
  • removeItem(productUuid): Removes the corresponding product from the cart. Keep updated on both arrays.
  • calculateTotal(): Calculates the total purchase value using the quantity and unit price of each product. The pricePerUnit is obtained from the internal product array (not from ProdList).

The showShoppingCart(fnDisplay, args) method is optional (extra points). It receives a function and its arguments to display the shopping cart. This method can be used to display the shopping cart on the HTML page. The arguments are optional and depend on the function received. Can be used to indicate where the shopping cart will be displayed. args can be an array or an object, depending on the function received.
